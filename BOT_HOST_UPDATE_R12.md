# Обновление Seller.pro.uz до r12 на BotHost

## Что загрузить

Загрузите в проект BotHost весь релизный ZIP с заменой файлов. Обязательно должны попасть:

- `main.py`
- `uzum_client.py`
- `uzum_finance.py`
- `formatters.py`
- `seller_pdf_report.py`
- `requirements.txt`
- `Dockerfile`
- `DejaVuSans.ttf`
- `DejaVuSans-Bold.ttf`

Не удаляйте постоянный каталог с `bot.db` и не заменяйте действующие переменные окружения токенами из примера.

## Новые рекомендуемые переменные

```text
UZUM_SKU_FINANCE_CACHE_SECONDS=600
UZUM_EXPENSE_MAX_PAGES=100
UZUM_EXPENSE_CACHE_SECONDS=300
LOGISTICS_REMINDER_CHECK_INTERVAL_SECONDS=1800
LOGISTICS_REMINDER_INVOICE_PAGES=5
LOGISTICS_REMINDER_RETURN_PAGES=5
```

Если их не добавлять, бот использует эти безопасные значения по умолчанию.

## Что проверить после перезапуска

В журнале должны появиться строки:

```text
PREMIUM_RELEASE_LOADED version=2026.07.20-premium-r12-uzum-finance-reminders
UZUM_FINANCE_LOADED: purchasePrice-only cost + expense ledger + IKPU audit
LOGISTICS_REMINDERS_LOADED: FBO slots + return paid-storage deadlines
PDF_FONT_READY: regular=/app/DejaVuSans.ttf bold=/app/DejaVuSans-Bold.ttf
```

Затем проверьте в Telegram:

1. **💰 Продажи → 💰 Прибыль** — источник себестоимости должен быть только Uzum.
2. **💰 Продажи → 🧾 Расходы Uzum** — показываются подтверждённые дополнительные расходы за 30 дней.
3. **📦 Склад → 🧾 ИКПУ / МХИК** — видны заполненные и отсутствующие коды.
4. **🌐 Все магазины** — отображается общий результат и строка каждого магазина.
5. **⚙️ Настройки → 🔔 Уведомления → 📦 Склад** — доступны переключатели срока поставки и возвратов.
6. **📊 Отчёты → 📄 PDF-отчёт** — PDF формируется на русском и узбекском.

Для автоматического расчёта налога один раз откройте **💰 Продажи → 🧮 Расходы и налоги** и укажите ставку конкретного магазина. Режим налогообложения Uzum API не передаёт, поэтому бот не угадывает процент. Расходы внутри Uzum загружаются автоматически; в настройках указываются только расходы вне Uzum.

## Как читается прибыль

- Себестоимость берётся только из поля `purchasePrice`, которое вернул Uzum.
- Если Uzum не передал себестоимость хотя бы для части проданных SKU, бот не подставляет цену продажи или среднее значение.
- Комиссия и логистика уже входят в расчёт выплаты и не вычитаются повторно из списка расходов.
- Поле называется **«Чистая прибыль»** только при полном покрытии и доступном API расходов. Иначе показан **«Результат по известным данным»** с предупреждением.

## Ограничение API

Текущая официальная схема Seller OpenAPI не передаёт надёжное поле бренда в используемом списке SKU. Поэтому в r12 добавлен аудит ИКПУ/МХИК, но бренд бот не выдумывает и не подставляет из названия товара.

## Откат

Перед обновлением создайте резервную копию командой `/backup_db`. Для отката верните предыдущие файлы приложения, но оставьте текущий `bot.db`: новые столбцы и таблицы совместимы со старой базой и не удаляют прежние ручные данные.
