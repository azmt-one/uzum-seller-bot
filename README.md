# Uzum Seller Assistant

Telegram-бот для селлеров Uzum: продажи, баланс, остатки, уведомления, отчёты, подписки и юнит-экономика.

## Релиз r11: стабильность и безопасность

- восстановлены команды `/products`, `/stock`, `/fbo`, `/fbs` и кнопка **«Все остатки»**;
- все статические кнопки русского и узбекского меню имеют ровно один обработчик;
- складские уведомления учитывают личные переключатели и порог каждого пользователя;
- `/trial TELEGRAM_ID ДНИ` теперь добавляет дни к действующему trial, а не сокращает его;
- `/backup_db` создаёт целостный SQLite-снимок вместе с данными WAL;
- Excel-импорт себестоимости ограничен по размеру и количеству строк и обрабатывается вне основного цикла бота;
- один пользователь не может параллельно запустить несколько тяжёлых обработчиков, а общая конкурентность ограничена;
- Docker-контекст исключает `.env`, базы, логи и другие локальные секреты;
- обновлена библиотека шифрования и добавлены регрессионные тесты.

## Управленческий PDF-отчёт

Пользователи с полной подпиской могут открыть **📊 Отчёты → 📄 PDF-отчёт**
или команду `/pdf_report` и выбрать период: сегодня, 7 или 30 дней. PDF содержит:

- ключевые продажи и динамику;
- прибыль с явным покрытием себестоимостью;
- остатки, прогноз дефицита и план действий;
- отмены, возвраты, новый и накопительный брак.

История нового брака ведётся ботом с момента установки этой версии. Docker-образ
устанавливает Unicode-шрифт автоматически; при локальном запуске без DejaVu Sans
можно указать путь к `.ttf` через `PDF_FONT_PATH`.

## Безопасный контроль подписок

Бот ежедневно проверяет `trial_until` и `subscription_until` в SQLite, создаёт
черновики за 7, 3 и 1 день до окончания и после окончания доступа, затем
присылает администраторам список действий. Черновики переживают перезапуск
бота и не дублируются.

- `/renewals` — обновить и открыть список действий;
- `/reminder НОМЕР` — проверить конкретный черновик;
- отправка клиенту выполняется только кнопкой **«Подтвердить и отправить»**;
- без подтверждения автоматизация не пишет клиентам, не продлевает подписку и
  не блокирует пользователя.

Время ежедневной админ-сводки задаётся через
`SUBSCRIPTION_ADMIN_DIGEST_HOUR_UZT` (по умолчанию 09:00, UTC+5), а окна —
через `SUBSCRIPTION_REMINDER_DAYS=7,3,1`.

## Запуск

1. Заполните переменные окружения на BotHost или создайте `.env` по примеру `.env.example`.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Запустите:

```bash
python main.py
```

Для BotHost загрузите содержимое релизного ZIP целиком, сохраните существующие
переменные окружения и постоянный каталог `/app/data`. Для полного поиска брака
проверьте значения `LOSS_REPORT_FILTERS=ALL,ARCHIVE,DEFECTED` и
`LOSS_REPORT_MAX_REQUESTS=60`. После запуска в журнале должна появиться строка:

```text
PREMIUM_RELEASE_LOADED version=2026.07.19-premium-r11-stability-security
```

Проверка перед публикацией:

```bash
python -m unittest -v test_subscription_automation.py test_release_regressions.py
```
